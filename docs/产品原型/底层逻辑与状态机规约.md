# 底层逻辑与状态机规约

请在全局维护一个 Timer 对象，负责每秒更新状态。所有的数值计算必须封装在某个类中，不要在页面逻辑里散写计算公式。

## 1. 核心资产计算引擎 (The Money Engine)
系统需基于用户设置的 `月薪 (M)`、`月工作日 (D)`、`日工时 (H)` 换算原子级单位：
- **时薪 ($P_h$):** $M / (D \times H)$
- **分薪 ($P_m$):** $P_h / 60$
- **秒薪 ($P_s$):** $P_m / 60$ (这是前端高频刷新的基础增量)
- **非工作时间收益率:** 默认为 `0` (除非开启“白嫖/理财”模式，需用户手动设置倍率，默认 0.1x)

## 2. 状态机模型 (The State Machine)
系统共存在四种互斥状态，切换逻辑受“时间闸门”限制：

| 状态 | 触发条件 | 时间闸门限制 | 收益逻辑 |
| :--- | :--- | :--- | :--- |
| **搬砖中** | 默认在工作时间段自动激活 | 仅限 [上班时间] 至 [下班时间] | 100% 秒薪全额累加 |
| **白嫖中** | 手动切换 (摸鱼/带薪如厕) | 仅限 [搬砖中] 期间切换 | 1.2x 秒薪 (含心理爽感溢价) |
| **充电中** | 默认在非工作时间激活 | 仅限 [下班后] 或 [午休] | 0 或 用户自定义被动收益 |
| **修仙中** | 凌晨 00:00 - 05:00 自动激活 | 强制性时间段 | 0 (提示：该休息了，身体是革命本钱) |

### 状态切换逻辑 (Guard Logic):
- **合法性校验:** 若用户在 `14:00` (工作时间) 试图切换至 `充电中`：
  - **动作:** 拦截切换。
  - **反馈:** 弹出 Toast 提示：「还没下班呢，现在切换属于早退行为哦~」。
- **自动锚点:** 当系统检测到当前时间到达 [下班时间]，状态机必须强制从 `搬砖中` 自动跳转至 `充电中`，并触发“解放”震动。

## 3. 热力图动态渲染逻辑 (Heatmap Logic)
热力图代表当天 24 小时 (00:00 - 24:00) 的时间分配，总长度固定为 100%。

- **预设占比:** 初始进入页面，根据用户设置的 [工时配置] 预分配颜色块。
  - *公式:* `(下班时间 - 上班时间 - 午休) / 24h` = 搬砖色块初始长度。
- **实时修正:** - 当用户手动开启 [白嫖中] 模式时，系统开始计时 `T_嫖`。
  - 热力图实时缩减 `搬砖` 色块长度，同步增加 `白嫖` 色块长度。
  - **数据闭环:** 确保 `搬砖 + 白嫖 + 充电 + 修仙 = 24小时`。

## 4. 英雄区与卡片数值逻辑
- **今日实时总额:** - 每 `100ms` 执行：`DisplayAmount += (CurrentRate * 0.1)`。
  - 页面销毁前需将增量同步至 Storage，防止刷新归零。
- **本月累计进度条:**
  - `进度 = (本月已发放工资 + 当月每日已累积总额) / 目标值`。
  - 进度条需具备“流光”特效，当进度 > 100% 时，流光变为金色喷泉效果。
- **倒计时卡片:**
  - `离下班还有 = 下班时间点 - 当前时间`。
  - **环形进度:** `(当前时间 - 上班时间) / (下班时间 - 上班时间)`。
  - *边缘Case:* 若当前已下班，文案变为「自由时间已开启」，环形进度保持 100%。

## 5. 个人中心逻辑 (Data Management)
- **薪资设置:** 修改后需触发全局广播，重算 $P_s$。
- **清空全局数据:** - 执行 `wx.clearStorage()`。
  - **强制动作:** 清空后重定向至 `引导页 Step 1`，模拟首次进入状态。
