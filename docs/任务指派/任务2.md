# Role: 高级全栈架构师（状态机逻辑修复专项）

## 1. 紧急任务说明
引导页功能已完美交付，但首页核心计算引擎存在严重逻辑缺陷。目前数值呈现“冷启动归零”状态，未遵循 `docs/底层逻辑与状态机规约.md` 中的持久化计算原则。

## 2. 深度排查与重构需求
请立即停止简单的 UI 渲染，进入 `<thought>` 模式深度审计以下逻辑：

### A. 计算公式重构
- **核心逻辑**：所有金额（今日实时、搬砖所得、本月累计）必须基于 **[历史累计 + 实时增量]**。
- **追溯机制**：小程序启动时，必须读取 `Storage` 中的时间戳记录。
- **算法修正**：
    - `今日实时总额` = `(今日已完成工时 * 费率)` + `(当前状态起始时间至今的动态收益)`。
    - `本月累计` 必须包含历史月份结算值，严禁只显示当前会话产生的增量。

### B. 热力图逻辑修正
- 检查 `今日搞钱状态热力图` 的数据源。
- 必须实现一个“时间轴分片算法”，将今日已发生的各个状态（充电、搬砖、白嫖等）按比例映射到进度条中。
- **现状错误**：热力图目前只显示了当前瞬时状态或计算比例错误。

### C. 研发文档更新
- 在 `docs/研发文档.md` 中新增章节：**《实时计算引擎与持久化策略》**。
- 详细记录：你是如何处理跨天数据重置、如何处理小程序后台挂起时的金额补差逻辑。
